# GALOHOo | AI-first game

## Русская версия

[Открыть игру](https://samayoucore.github.io/GALOHOo-AI-first-game/)

**GALOHOo** — браузерная 2D-игра с пиксельной графикой и видом сверху, созданная как эксперимент в области AI-first разработки игр.

Основная идея проекта — связать реальную продуктивность игрока с развитием виртуального мира. Игрок создаёт задачи в ToDo-листе, указывает их важность, выполняет их в реальной жизни и получает за это игровое золото и ресурсы. Полученные награды можно использовать для развития небольшого поселения на острове.

## Концепция

GALOHOo объединяет инструмент для управления задачами и уютный симулятор поселения.

Вместо выполнения заранее подготовленных игровых заданий игрок продвигается за счёт собственных реальных дел. ToDo-лист становится основным источником золота, материалов и долгосрочного развития игрового мира.

Действие происходит на случайно сгенерированном острове. На нём живут управляемые системой жители, которые самостоятельно перемещаются, собирают ресурсы, отдыхают, строят здания и взаимодействуют с окружением.

Главная цель игрока — развивать поселение, совмещая выполнение реальных задач с управлением ресурсами и жизнью острова.

## Цель проекта

GALOHOo — экспериментальная productivity-игра, которая превращает повседневные задачи в часть игровой экономики.

Проект исследует простой вопрос:

Связывая выполнение реальных задач с развитием виртуального поселения, GALOHOo делает продуктивность более наглядной, вознаграждающей и игровой.

## English Version

[Play the game](https://samayoucore.github.io/GALOHOo-AI-first-game/)

**GALOHOo** is a browser-based 2D pixel-art game with a top-down view, created as an experiment in AI-first game development.

The main idea of the project is to connect the player’s real-life productivity with the development of a virtual world. Players create tasks in a ToDo list, assign an importance level, complete them in real life, and receive in-game gold and resources as a reward. These rewards can then be used to develop a small island settlement.

## Concept

GALOHOo combines a task-management tool with a cozy settlement simulation game.

Instead of progressing through predefined quests, players advance by completing their own real-life tasks. The ToDo list becomes the main source of gold, resources, and long-term development within the game world.

The game takes place on a randomly generated island inhabited by system-controlled villagers. They move around independently, collect resources, rest, construct buildings, and interact with the environment.

The player’s main goal is to develop the settlement by combining real-life productivity with resource management and control over the island’s growth.

## Main Features

- browser-based 2D pixel-art game;
- top-down island view;
- random island generation on the first launch;
- ToDo list as the main gameplay mechanic;
- task importance system;
- in-game rewards for completing real-life tasks;
- gold and resource-based economy;
- shared settlement storage containing wood, stone, and other materials;
- villagers with individual behavior;
- automatic resource collection by villagers;
- randomly generated villager profiles and characteristics;
- villager professions based on their actual activities;
- energy, work, fatigue, and sleep systems;
- day and night cycle;
- weather effects, including rain;
- settlement development through ToDo-based quests;
- era progression that changes buildings and villager appearances;
- building suggestions made by villagers;
- the ability to approve or reject construction;
- a traveling merchant who periodically arrives on the island;
- a gacha system for spending extra gold;
- unlockable custom cursors;
- the ability to pick up and drag villagers;
- special animations for dragging characters and dropping them into the water.

## AI-first Development

The project was created using an AI-first development workflow.

Most of the codebase was generated with Google Gemini. My work on the project included:

- developing the game concept;
- designing gameplay mechanics and systems;
- defining the logic connecting different mechanics;
- writing and improving prompts;
- planning new features iteratively;
- making UI and UX decisions;
- testing the gameplay;
- identifying, documenting, and verifying bug fixes;
- making small manual HTML and CSS adjustments;
- polishing the final user experience.

The goal of the experiment was not only to create a playable browser game, but also to explore how far an AI-assisted workflow can go when building a game prototype — from the initial idea to a working implementation.

## Project Goal

GALOHOo is an experimental productivity game that transforms everyday tasks into part of a game economy.

The project explores a simple question:

> **Can a ToDo list feel like a living world instead of a boring checklist?**

By connecting real-life task completion with the development of a virtual settlement, GALOHOo aims to make productivity more visual, rewarding, and playful.
